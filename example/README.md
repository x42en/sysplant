# SysPlant — Injection Examples

This directory contains working injection examples for each supported language.
Each example performs the same operation: **process hollowing** into a sacrificial process (`notepad.exe`) using direct syscalls generated by SysPlant.

> **⚠️ Disclaimer**: These examples are provided for **educational and authorized security research purposes only**. The included shellcode launches `calc.exe` as a harmless proof of concept.

---

## Prerequisites

### SysPlant

All examples require SysPlant to generate the syscall file first. Install it following the [main documentation](../README.md), then generate the appropriate syscall file for your target language.

```bash
# Install sysplant (if not already done)
pip install sysplant
# or from source
cd /path/to/sysplant && pip install .
```

---

## 1. C Example

### Environment Setup

Install the MinGW cross-compiler for Windows targeting:

```bash
sudo apt install mingw-w64
```

### Generate Syscall File

```bash
# Generate syscall.h next to inject.c
python -m sysplant generate -c -o example/syscall.h canterlot
```

### Compile

```bash
x86_64-w64-mingw32-gcc -Wall -s -static -masm=intel example/inject.c -o example/inject.exe
```

### Run

Transfer `inject.exe` to a Windows machine and execute it:

```cmd
inject.exe
```

## 2. Nim Example

### Environment Setup

Install Nim and the required `winim` library:

```bash
# Install Nim (https://nim-lang.org/install.html)
curl https://nim-lang.org/choosenim/init.sh -sSf | sh

# Install winim bindings
nimble install winim

# MinGW is also required for cross-compilation
sudo apt install mingw-w64
```

### Generate Syscall File

```bash
# Generate syscall.nim next to inject.nim
python -m sysplant generate -nim -o example/syscall.nim canterlot
```

### Compile

```bash
nim c -d=release -d=danger -d=strip --opt=size -d=mingw --app=console --cpu=amd64 --out=example/inject.exe example/inject.nim
```

### Run

Transfer `inject.exe` to a Windows machine and execute it:

```cmd
inject.exe
```

## 3. Rust Example

### Environment Setup

Install Rust and configure cross-compilation for Windows:

```bash
# Install Rust (https://rustup.rs)
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# Add Windows GNU target
rustup target add x86_64-pc-windows-gnu

# MinGW is required as linker
sudo apt install mingw-w64
```

Create a Cargo project and configure dependencies:

```bash
cargo init --name inject example/rust-inject
```

Edit `example/rust-inject/Cargo.toml` to add the required dependencies:

```toml
[package]
name = "inject"
version = "0.1.0"
edition = "2024"

[dependencies]
md5 = "0.7"
windows-sys = { version = "0.59", features = [
    "Win32_System_SystemServices",
    "Win32_System_SystemInformation",
    "Win32_System_Diagnostics_Debug",
    "Win32_System_Kernel",
    "Win32_System_Threading",
    "Win32_System_Memory",
    "Win32_Security",
    "Win32_Foundation",
]}
```

Configure the linker for cross-compilation by creating `example/rust-inject/.cargo/config.toml`:

```toml
[target.x86_64-pc-windows-gnu]
linker = "x86_64-w64-mingw32-gcc"
```

### Generate Syscall File

```bash
# Generate syscall.rs into the Cargo project src/ directory
python -m sysplant generate -rust -o example/rust-inject/src/syscall.rs canterlot
```

Then copy the example file as the project entry point:

```bash
cp example/inject.rs example/rust-inject/src/main.rs
```

### Compile

```bash
cd example/rust-inject
cargo build --release --target x86_64-pc-windows-gnu
```

The compiled binary will be at `target/x86_64-pc-windows-gnu/release/inject.exe`.

### Run

Transfer `inject.exe` to a Windows machine and execute it:

```cmd
inject.exe
```

## Options

All `sysplant generate` examples above use the `canterlot` iterator as a reference, but you can substitute any available iterator:

| Iterator       | Description                               |
| -------------- | ----------------------------------------- |
| `hell`         | Hell's Gate technique                     |
| `halo`         | Halo's Gate technique                     |
| `tartarus`     | Tartarus' Gate technique                  |
| `freshy`       | FreshyCalls technique                     |
| `syswhispers`  | SysWhispers2 technique                    |
| `syswhispers3` | SysWhispers3 technique                    |
| `canterlot`    | Canterlot technique (exception directory) |

Additional useful flags:

```bash
# Scramble internal function names (recommended for evasion)
python -m sysplant generate -rust -x -o syscall.rs canterlot

# Use indirect syscall caller stub
python -m sysplant generate -rust -o syscall.rs canterlot indirect

# Select specific syscall functions instead of the default common set
python -m sysplant generate -rust -f NtOpenProcess,NtAllocateVirtualMemory -o syscall.rs canterlot
```

## File Structure

After generating all examples, the directory should look like:

```
example/
├── README.md           # This file
├── inject.c            # C injection example
├── inject.nim          # Nim injection example
├── inject.rs           # Rust injection example (copy to rust-inject/src/main.rs)
└── rust-inject/        # Cargo project (created by user)
    ├── Cargo.toml
    ├── .cargo/
    │   └── config.toml
    └── src/
        ├── main.rs     # Copy of inject.rs
        └── syscall.rs  # Generated by sysplant
```
